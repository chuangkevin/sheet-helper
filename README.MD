# Sheet Helper

Google Sheets 車輛庫存管理整合工具，用於同步車源表、庫存表，並與 `car-prompts` 專案整合產生 PO 文。

## 功能

- 讀取 Google Sheets 的「車源」和「庫存」工作表
- 自動判斷車輛狀態（依據背景顏色）
- 整合兩表資料，建立「整合庫存」工作表
- 解析內裝色欄位，分離顏色和改裝資訊
- 設定下拉選單方便篩選
- 與 `car-prompts` 資料夾同步 PO 狀態

---

## 快速開始

### 安裝

```bash
npm install
```

### 設定

1. 複製環境變數範例：
   ```bash
   cp .env.example .env
   ```

2. 編輯 `.env`，填入你的 Google Sheets ID

3. 設定 Google OAuth 憑證：
   - 前往 [Google Cloud Console](https://console.cloud.google.com/)
   - 建立專案並啟用 **Google Sheets API**
   - 建立 OAuth 2.0 憑證（桌面應用程式）
   - 下載 JSON 並重新命名為 `credentials.json` 放到專案根目錄
   - 在 OAuth 同意畫面 → 目標對象，加入你的 email 為測試用戶

4. 首次執行授權：
   ```bash
   npm run dev
   ```
   瀏覽器會開啟 Google 登入頁面，授權後會自動產生 `token.json`

### 執行

```bash
# 建立/更新整合庫存表
npx ts-node src/create-integrated-sheet.ts

# 檢查庫存和 PO 狀態
npx ts-node src/check-status.ts

# 調整欄寬
npx ts-node src/fix-width.ts
```

## 表格結構

### 車源表

- **工作表名稱**: `車源`
- **資料起始行**: Row 2（Row 0 空白，Row 1 標題）

#### 欄位索引（0-indexed）

| Index | 欄位 | 說明 |
|-------|------|------|

## 規則與邏輯

### 背景顏色 → 狀態判斷

| 顏色 | 狀態 | 說明 |
|------|------|------|
| 🔴 紅色 | 已售出 | R>0.7, G<0.5, B<0.5 |
| ⬜ 白色 | 在庫 | R>0.95, G>0.95, B>0.95 |
| 🟨 淺黃 | 新到貨 | RGB(255,242,204) |
| 🟣 紫紅 | 特殊 | RGB(255,0,255) |

### Item 編號 → 來源判斷

| 前綴 | 來源 | 範例 |
|------|------|------|
| 純數字 | 國外進口 | 2, 39, 117 |
| P | 國外進口 | P8, P11 |
| T | 託售 | T6, T12 |
| A | 台灣車 | A54, A99 |
| B | 寄賣 | B151, B152 |

### 分隔標記行

表格中有特殊分隔行需跳過：
- `台灣` - 台灣車區塊
- `寄賣` - 寄賣車區塊

### 車況評分格式

格式：`外觀分數/內裝等級`

| 評分 | 說明 |
|------|------|
| 6/A | 最佳 |
| 5/A | 優良 |
| 4.5/B | 中上 |
| 4/C | 較差 |

### 內裝色解析

內裝色欄位可能包含改裝資訊，需分離：

| 原始值 | 內裝色 | 改裝 |
|--------|--------|------|
| `黑(星空頂)` | 黑 | 星空頂 |
| `米(ACC/米勒)` | 米 | ACC/米勒 |
| `紅 Mansory` | 紅 | Mansory |
| `黑/白` | 黑/白 | (無) |

#### 常見改裝項目
- 音響、柏林之音
- ACC（主動巡航）、米勒（抬頭顯示）
- 星空頂、私密隔屏、冰箱
- 碳纖維、黑化、Mansory
- 四座、五座、七座

## 注意事項

1. **敏感檔案**: `credentials.json`、`token.json`、`.env` 不可提交到 Git
2. **測試用戶**: OAuth 應用程式處於測試階段時，需在 Google Cloud Console 加入測試用戶
3. **API 限制**: Google Sheets API 有請求頻率限制
4. **資料同步**: 整合庫存表是合併產生，修改後不會回寫原表
5. **VIN = 引擎碼**: 此專案中 VIN 就是引擎碼，不需要額外解碼
